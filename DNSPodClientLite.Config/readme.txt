== DNSPodClientLite介绍 ==
DNSPodClientLite是DNSPod客户端的简化版本，目前提供动态域名解析和宕机监控的功能。

== 使用介绍（使用前必读） ==
1. 本客户端操作比较简单，动态域名和宕机监控的功能都是在记录列表上右键选择相应的操作就可以了。
2. 图文使用帮助：https://support.dnspod.cn/Kb/showarticle/tsid/19

== 常见问题 （遇到问题可以看这里）==
* 客户端一直都获取不到最新IP，并且日志里大量报错怎么办？
 * 先手工看telnet ns1.dnspod.net 6666能否成功，如不成功，需要设置防火墙允许访问对外的6666端口。

* 我的服务器是双线的，一条线路是动态IP，一条线路是静态IP该怎么办？
 * 首先通过ping，得到ns1.dnspod.net和dnsapi.cn的ip地址
 * 在路由设置静态路由，让到这两个IP的数据都走动态IP所在的WAN口

* 我需要一直开着客户端吗？
 * 不需要，把本程序安装成服务后，关掉客户端也可以进行动态解析和宕机监控。

* 我的服务器自动重启后，客户端会自动运行吗？
 * 是的，把本程序安装成服务后，程序会一直运行，即使服务器意外重启，该客户端也会自动启动。

* 监控切换时可否支持智能选择备用服务器？
 * 不支持，本客户端只支持指定IP作为备用服务器，要使用智能切换请使用网页版宕机监控。

* 宕机监控里设置的备用服务器IP是否可以作为另一条记录的值？
 * 可以，如果你有两个IP，可以添加两条记录，并且互为备份，两个IP同时宕机的可能性应该很小。

* 本客户端在宕机时可否给我发送邮件和短信提醒？
 * 不可以，如需要该功能，请使用网页版宕机监控功能。

* 我本机监控了一条记录，如果我在网站上删除了该条记录，如何删除本地的监控？
 * 在客户端上打开这个域名的记录列表，客户端会自动删除服务器上删除掉的记录。

* 我本机监控了一条记录，如果我在网站上修改了这条记录的IP或子域名怎么办？
 * 需手工在客户端上删除该条记录的监控并重新添加。

* log目录下已经有很多日志了，我可以清理吗？
 * 可以清理掉log下的日志文件，但服务运行时log目录不要删掉。
